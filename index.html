CRUD 应用服务端的替代解决方案探索

5月 05, 2016 in 服务端

去年底做 Android 客户端开发的时候，服务端的开发进度始终跟不上，就找了一个测试服务器框架 Moco, 自己搭建测试服务器生成 Json 格式的响应，用来测试 Android 代码。

后来在涉及到创建和删除数据的时候，就没法再继续使用 Moco 的 API 来测试了。第一想法是要不自己做个简易的 Java EE 服务端，但看到 Spring MVC 有点头大，感觉有点小题大做。去 Sring 官网逛了一圈，发现了 SpringBoot, 自带 Servlet 容器，运行 Web 运用就像运行一个本地 Java 应用，感觉很方便，把玩了一下觉得还是很不错，简单多了。

用了用依然还是嫌麻烦，只打算做个 App，还要边做边弄 一堆后端的内容，想想就闹心。为了偷懒，继续搜寻了一番，发现了 Parse 和 LeanCloud 等 PaaS 平台，对于大部分 CRUD 应用，使用平台自带的 API 已经足够。考虑到网络原因，我选用的 LeanCloud, API 基本跟 Parse 一模一样，控制台的用户体验稍微有点差强人意，自带的 _User 类使用起来有些小 BUG, 写了个应用测试了一下觉得访问速度还行，凑合着先用到。

没想到年后过来想使用的时候，发现 LeanCloud 上之前测试时建立的数据已经被清空了，今天刚好看到 LeanCloud 官方发来的安全风险提示邮件，不清楚是不是由于安全方面的漏洞导致的。所以说数据放在别人手里，始终还是有风险，无论是出于安全性还是稳定性的考虑。

之前一直关注着 Parse, 1月底的时候知晓 Facebook 已将 Parse开源，整体业务在6月结束，官网之前有一份公告，提示用户将所有应用的数据进行迁移，同时给出了详尽的建立本地 Server 的流程和迁移指南，于是决定在本地建立 Parse-Server，用来作为 App 的后端。

Mac 使用心得

5月 03, 2016 in Mac

感谢老婆送的生日礼物 MacBook Pro，使用快两个月了，基本上每天都会高呼一遍『 MacBook 真的太好用啦！』^_^

之前曾数次将老笔记本和 PC 装上 Ubuntu，但几个回合下来，臣妾真的做不到啊……一个 NVIDIA 的显卡驱动装了一天未果，面对着各种拖影和闪烁欲哭无泪。现在回想起来，可能是因为我并没有摆脱 Windows 下的操作习惯，首先想的就是先整个 GUI 安抚一下自己忐忑的玻璃心。

然后就是被各种 Shell 命令折腾，只记住了 apt-get 相关的一些，还有就是个 sudo……

拿到 MacBook 后，一开始也抓瞎了好多天，但是每次看到那精致的设计，那极其符合审美口味的界面，都告诉自己一定要尽快把它玩转。

接下来就是通过各种途径搜集学习扫盲的过程，知乎上观摩各路大神的经验之谈，在综合参考了以下几个问题的回答后

  • Mac 下有什么软件能够极大地提高工作效率？
  • OS X 上有哪些优质的 app？
  • 程序员如何优雅地使用 Mac？

同时结合了自己的常用操作习惯，循序渐进用了大概一个月后，总结出来的一些内容

  • Alfred

这个必须毫无争议地位列神器排行榜第一，也是在我看来从 Windows 操作系统转到 OS X 后应该第一时间了解和学习的软件。

将这个作为切入点有助于首先放空 Windows 在头脑中建立多年的根深蒂固的操作习惯，从用鼠标点击桌面/开始菜单图标的习惯过渡到通过输入文字进行应用程序的启动、文件的读取、网页的开启，这是一个非常有趣的转变，一开始可能会觉得有一点不习惯，但是不需要多久，就会发现这种方式不知道比拿起鼠标——寻找桌面图标——点击图标这过程快了多少倍，尤其是在桌面乱如麻的盛况下。这个细微的习惯的转变对于小白用户后期认识和学习 Shell 也是一个非常好的打基础的过程。

Alfred 如果需要使用 Workflow 的话需要收费，我现在暂时没有深入把玩，已经觉得很有用了。OS X 自带的 Sportlight 和 Alfred 性质一样但扩展性不如后者，我个人感觉对拼音模糊搜索的功能比 Alfred 更好一点。

在习惯这种操作方式之后，已经停不下来了。经过寻觅，发现在 Windows 下有类似的软件 Wox, 果断入手，之前的版本 BUG 颇多且在 Win10 下经常崩溃，最新的貌似已经修复。

  • CheatSheet

CheatSheet 主要的作用是在新接触 OS X 的时候，可以随时通过长按 ⌘ command 键来查看当前环境中的快捷键。

在使用过程中发现，对于像 Safari 等自带的软件和运行环境支持较好，但在某些 IDE 中，只能显示部分快捷键，不过对于新手来说，也是完全够用了。

  • Vimari

使用 Vimari 的原因是因为不愿意插上鼠标使用 MacBook，只想用键盘和触控板完成所有操作。Vimari 用于 Safari, Chrome 上面的同类型插件是 Vimium. 由图片可以看出，它们的作用是将浏览器的交互区域映射至键盘，使用类似于 Vim 快捷键的方式来进行操作。乍一看感觉很麻烦，但在习惯后这种操作流程完全是行云流水，比用鼠标高效不少，在一定程度上也让我开始理解了 Vim 的设计思想。

用WSDL生成RxAndroid结构的安卓客户端代理类

4月 10, 2016 in Android

服务端采用 WCF 时，.net 的开发环境可以根据 WSDL 文件直接生成客户端代理类，很方便地用类似调用本地方法的方式访问服务。
但是基于 JAVA 的安卓开发环境下就只有采用第三方的工具了，在网上查找 CXF 和 Axis 等相关技术后，发现由于底层 JVM 和 Dalvik 的内部差异，生成的 JAVA 客户端代理类在安卓开发环境中并不能使用。
经过多方寻觅，终于发现一个国外的好东东—————Wsdl2Code

看简介

Wsdl2Code is a web tool that helps developers consume .net web services.

Simply insert your Service URL or upload a WSDL file and that’s it.

The engine will generate the classes and methods for connecting Soap Services From Android and iOs in seconds!

Android generated code works using Ksoap2

iOs generated code works with Xpathquery , libxml library and NSData+Base64 For Byte Array Support
