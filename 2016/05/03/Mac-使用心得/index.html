Mac 使用心得

5月 03, 2016 in Mac

感谢老婆送的生日礼物 MacBook Pro，使用快两个月了，基本上每天都会高呼一遍『 MacBook 真的太好用啦！』^_^

之前曾数次将老笔记本和 PC 装上 Ubuntu，但几个回合下来，臣妾真的做不到啊……一个 NVIDIA 的显卡驱动装了一天未果，面对着各种拖影和闪烁欲哭无泪。现在回想起来，可能是因为我并没有摆脱 Windows 下的操作习惯，首先想的就是先整个 GUI 安抚一下自己忐忑的玻璃心。

然后就是被各种 Shell 命令折腾，只记住了 apt-get 相关的一些，还有就是个 sudo……

拿到 MacBook 后，一开始也抓瞎了好多天，但是每次看到那精致的设计，那极其符合审美口味的界面，都告诉自己一定要尽快把它玩转。

接下来就是通过各种途径搜集学习扫盲的过程，知乎上观摩各路大神的经验之谈，在综合参考了以下几个问题的回答后

  • Mac 下有什么软件能够极大地提高工作效率？
  • OS X 上有哪些优质的 app？
  • 程序员如何优雅地使用 Mac？

同时结合了自己的常用操作习惯，循序渐进用了大概一个月后，总结出来的一些内容

  • Alfred

这个必须毫无争议地位列神器排行榜第一，也是在我看来从 Windows 操作系统转到 OS X 后应该第一时间了解和学习的软件。

将这个作为切入点有助于首先放空 Windows 在头脑中建立多年的根深蒂固的操作习惯，从用鼠标点击桌面/开始菜单图标的习惯过渡到通过输入文字进行应用程序的启动、文件的读取、网页的开启，这是一个非常有趣的转变，一开始可能会觉得有一点不习惯，但是不需要多久，就会发现这种方式不知道比拿起鼠标——寻找桌面图标——点击图标这过程快了多少倍，尤其是在桌面乱如麻的盛况下。这个细微的习惯的转变对于小白用户后期认识和学习 Shell 也是一个非常好的打基础的过程。

Alfred 如果需要使用 Workflow 的话需要收费，我现在暂时没有深入把玩，已经觉得很有用了。OS X 自带的 Sportlight 和 Alfred 性质一样但扩展性不如后者，我个人感觉对拼音模糊搜索的功能比 Alfred 更好一点。

在习惯这种操作方式之后，已经停不下来了。经过寻觅，发现在 Windows 下有类似的软件 Wox, 果断入手，之前的版本 BUG 颇多且在 Win10 下经常崩溃，最新的貌似已经修复。

  • CheatSheet

CheatSheet 主要的作用是在新接触 OS X 的时候，可以随时通过长按 ⌘ command 键来查看当前环境中的快捷键。

在使用过程中发现，对于像 Safari 等自带的软件和运行环境支持较好，但在某些 IDE 中，只能显示部分快捷键，不过对于新手来说，也是完全够用了。

  • Vimari

使用 Vimari 的原因是因为不愿意插上鼠标使用 MacBook，只想用键盘和触控板完成所有操作。Vimari 用于 Safari, Chrome 上面的同类型插件是 Vimium. 由图片可以看出，它们的作用是将浏览器的交互区域映射至键盘，使用类似于 Vim 快捷键的方式来进行操作。乍一看感觉很麻烦，但在习惯后这种操作流程完全是行云流水，比用鼠标高效不少，在一定程度上也让我开始理解了 Vim 的设计思想。

  • oh-my-zsh

还记得之前在 Ubuntu 上痛苦地使用 Bash 的场景，那时候根本没想到 Shell 也可以用得如此帅气。OS X 自带多种 Shell，Zsh 是其中的一种，以强大和难以配置著称。但有了 oh-my-zsh，轻松使用已不是梦，一切都可以通过编辑 .zshrc 来进行配置。

我现在只是将配色改了一下，添加了一堆 alias, 只使用到其中很少的一部分,其他的特色功能可以参考使用zsh的十大优点。

在 Windows 下可以使用 Cygwin + babun 来体验，babun 中内置了 Zsh 和 oh-my-zsh.

网上还有很多人推荐 Fish, 在官网看了一下貌似支持命令的提示和预览，看着也很酷炫，以后再来折腾。
