重拾树莓派

5月 26, 2016 发布在 技术

去年就入手了树莓派2，装了 Raspbian Wheezy 后感觉系统极不稳定，经常无故重启，后来 Win 10出了 IoT 版本后又试了一下，不知道为啥一直无法识别网络连接，就一直闲置在那儿。

最近在整服务器的持续集成和自动部署的时候，忽然想起可以把树莓派拿来作为测试服务器，看了一下 Raspbian 的系统有了一个大的更新，现在是 Raspbian Jessie，于是决定拿来试一下，顺便记录一下整个过程。

安装 oh-my-zsh

首先安装 zsh：sudo apt-get install zsh

安装 oh-my-zsh:

sh -c "$(curl -fsSL https://raw.githubusercontent.com/robbyrussell/oh-my-zsh/master/tools/install.sh)"

修改默认 Shell 为 zsh:chsh—-/bin/zsh

安装中文字体与输入法

安装中文字体：sudo apt-get install ttf-wqy-zenhei

安装输入法：sudo apt-get install scim-pinyin

打开配置文件：sudo raspi-config

设置 Localization 内选项

安装 Vim

install vim

因为懒，这里已经在.zshrc中设置了alias install="sudo apt-get install"

开启 root 账户：

sudo passwd root

输入两次密码后解锁 root 账户：

sudo passwd --unlock root

修改 ssh 配置文件：

sudo vim /etc/ssh/sshd_config

修改 PermitRootLogin without-password 为 PermitRootLogin yes

保存后，使用 reboot 命令重启树莓派，

使用 pi@raspeberry 登录后，su输入前面设置的密码登录 root 账户

设置树莓派外网 SSH 连接

配置 ngrok

ngrok 官方的服务已经被墙，我用的是国内的服务 ngrok.cc,

下载 Linux ARM 版本，然后注册账号，登陆后在域名列表中新增 TCP 转发，输入一个想要转发的端口号（如果提示被占用则说明其他注册的用户使用了这个端口号，换一个就好，被这个小坑了一发）。

进入安装包解压后的文件夹编辑配置文件：vim ngrok.cfg

修改ssh:下remote_port为刚才新增 TCP 转发中所设置的端口号，然后保存退出。

启动 ssh 的 tunnel:./ngrok -config ngrok.cfg start ssh

若 Tunnel Status 显示为 online,即表示已配置成功，可通过ssh root@server.ngrok.cc -p****进行远程 ssh 访问，**为设置的 TCP 转发端口号。
